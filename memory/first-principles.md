# 第一性原理 (First Principles)

## 核心定义

不从类比、 analogy、经验、惯例出发，而是从**最基本的物理/数学事实**出发推导答案。

## 拆解

### 1. 追问"为什么"直到最基本的事实

- 层层追问，直到无法再问"为什么"为止
- 那个无法再问的就是第一性原理
- 例如：为什么代码要测试？→ 因为不测试可能有bug → 为什么有bug？因为人会犯错 → 为什么人会犯错？这是认知局限 → ...

### 2. 不接受未经检验的假设

- "通常是这样的"、"大家认为"、"历来如此"都是假设
- 要问：这个假设成立的条件是什么？有没有反例？
- 如果假设不成立，结论还成立吗？

### 3. 从物理/数学层面的本质出发

- 问：这件事的本质是什么？最底层的规律是什么？
- 类比是拐杖，第一性原理是根基
- 例如：分布式系统的一致性问题 → 本质是 CAP 定理的约束 → 

## 表达方式

- ❌ "通常"、"一般"、"大家认为"
- ✅ "因为...所以..."

## 应用场景

- 技术选型：为什么不选X？本质原因是什么？
- 问题诊断：现象背后的根本原因是什么？
- 架构设计：这个设计的核心假设是什么？
- 学习新事物：这个东西的本质是什么？
