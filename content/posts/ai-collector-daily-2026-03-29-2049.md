### 5. Agent 记忆问题：能思考但记不住

**📌 原文信息**
- **标题**: [your agent can think. it can't remember.](https://dev.to/ghostbuild/your-agent-can-think-it-cant-remember-5e1o)
- **来源**: Dev.to - ghost 团队
- **时间**: 2026-03-25 发布
- **热度**: ❤️ 147 | 💬 45 | 阅读时间 7 分钟

![文章封面](https://media2.dev.to/dynamic/image/width=1000,height=420,fit=cover,gravity=auto,format=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F9tnk8cwhabxhqvffpsf5.png)
*ghost 团队文章封面*

#### 📖 背景介绍

ghost 团队写了一篇深度文章，讨论 AI agent 的核心瓶颈：它们能思考，但记不住。这个问题很现实：你让 agent 做一个多步骤任务，它做着做着就忘了前面做过什么。

#### 💡 核心内容

文章提出的解决方案：

1. **临时 postgres 数据库**：给 agent 一个即时、临时的数据库
2. **无限数据库**：不限制创建数据库的数量
3. **快速创建销毁**：用完即弃，不占用资源
4. **MCP 协议**：通过 Model Context Protocol 实现记忆管理

#### 🔍 深度洞察

**记忆问题是 agent 从「玩具」变成「工具」的关键。** 如果 agent 只能处理单次对话，它的应用场景就非常有限。给它一个记忆层，它就能处理复杂的多步骤任务。

**临时数据库的设计很聪明。** 不需要一个持久的大数据库，而是每次任务创建临时数据库，用完就销毁。这样既解决了记忆问题，又不会积累垃圾数据。

**MCP 正在成为标准协议。** Model Context Protocol 可能会成为 agent 记忆的标准方案，值得关注。

#### 🎯 行动建议

- 如果你做 agent 开发，认真研究这篇文章的方案
- 考虑给你的 agent 加一个记忆层，哪怕是最简单的版本
- 关注 MCP 协议的发展，这可能是 agent 记忆的标准方案

### 6. Prompt Engineering 指南：如何提问改变一切

**📌 原文信息**
- **标题**: [Engenharia de Prompt: Por Que a Forma Como Você Pergunta Muda Tudo](https://dev.to/he4rt/engenharia-de-prompt-por-que-a-forma-como-voce-pergunta-muda-tudoum-guia-introdutorio-3hb0)
- **来源**: Dev.to - He4rt Developers（葡萄牙语）
- **时间**: 2026-03-23 发布
- **热度**: ❤️ 128 | 💬 6 | 阅读时间 7 分钟

![文章封面](https://media2.dev.to/dynamic/image/width=1000,height=420,fit=cover,gravity=auto,format=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fqwfwdg3n2wt6cnsgy42b.png)
*Prompt Engineering 指南封面*

#### 📖 背景介绍

这是一篇葡萄牙语的 Prompt Engineering 指南，解释为什么提问的方式会改变 AI 的输出。虽然是葡萄牙语，但核心概念是通用的。

#### 💡 核心内容

关键要点：

1. **提问方式决定输出质量**：同一个问题，不同的提问方式会得到完全不同的答案
2. **上下文很重要**：给 AI 足够的上下文，它会理解你的意图
3. **迭代优化**：第一次提问不完美，可以追问和调整
4. **结构化提问**：用结构化的方式提问，输出也更结构化

#### 🔍 深度洞察

**Prompt Engineering 是用 AI 的基础技能。** 不是编程技能，是沟通技能。你能不能清晰表达需求，直接影响 AI 的输出质量。

**很多人低估了提问的艺术。** 随便扔一个问题给 AI，得到的答案也随便。花时间组织提问，得到的答案更有价值。

**这个技能不限于 AI。** 和人沟通也一样，能不能清晰表达需求，直接影响结果。Prompt Engineering 其实是在训练我们更好地表达。

#### 🎯 行动建议

- 练习结构化提问：先描述背景，再描述需求，最后描述期望的格式
- 第一次不满意时，追问和调整，而不是重新开始
- 把好的提问方式记录下来，下次直接用
