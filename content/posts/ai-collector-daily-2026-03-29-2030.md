![V2EX](https://cdn.v2ex.com/navatar/c20a/d4d7/12_large.png?m=1751718333)
*V2EX 问与答板块*

#### 📖 背景介绍

楼主在 V 站发帖，说现在刷站发现「所有东西都在跟 AI 沾边」，所以很想知道有没有人真的用 AI 赚到过钱，哪怕是一块钱。

这个问题的背景是：AI 热潮持续了这么久，真正落地的变现案例到底有多少？很多人在用 AI，但靠 AI 赚钱是另一回事。

#### 💡 核心内容

评论区有几个真实的变现案例分享：

1. **AI 代写**：有人接商业文案代写，用 AI 生成初稿再人工润色，效率翻三倍
2. **AI 绘图接单**：有人做定制头像、logo 设计，用 Midjourney 或 Stable Diffusion 出图
3. **AI 数据分析**：帮企业做数据分析报告，用 AI 处理数据生成洞见

#### 🔍 深度洞察

这个帖子揭示了几个关键点：

**赚钱的不是 AI 本身，而是用 AI 解决问题的能力。** 会用 AI 生成文字不赚钱，但知道谁需要文字、需要什么样的文字，这个信息差才值钱。

**AI 是工具，不是生意。** 很多人把 AI 当成「赚钱机器」，但真正的赚钱逻辑是：发现需求 → 用 AI 高效解决 → 收费。AI 只是中间的工具环节。

**门槛在降低，竞争在加剧。** 当所有人都会用 AI 生成内容时，单纯的「AI 生成」就没有溢价了。差异化在于：懂行业、懂需求、能交付。

#### 🎯 行动建议

- 如果你会编程，可以考虑做 AI 工具或插件，解决具体场景问题
- 如果你不擅长技术，可以研究 AI 在自己行业的应用，比如 AI 辅助写作、AI 辅助设计
- 关注「AI + 传统行业」的交叉点，那里机会更多

### 4. Agent 记忆问题：能思考但记不住

**📌 原文信息**
- **标题**: [your agent can think. it can't remember.](https://dev.to/ghostbuild/your-agent-can-think-it-cant-remember-5e1o)
- **来源**: Dev.to - ghost 团队
- **时间**: 2026-03-25
- **热度**: ❤️ 147 | 💬 45

![cover](https://media2.dev.to/dynamic/image/width=1000,height=420,fit=cover,gravity=auto,format=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F9tnk8cwhabxhqvffpsf5.png)
*文章封面*

#### 📖 背景介绍

ghost 团队写了一篇深度文章，讨论 AI agent 的核心瓶颈：它们能思考，但记不住。

这个问题很现实：你让 agent 做一个多步骤任务，它做着做着就忘了前面做过什么。原因是大多数 agent 没有持久的记忆层。

#### 💡 核心内容

文章提出的解决方案：

1. **临时 postgres 数据库**：给 agent 一个即时、临时的数据库
2. **无限数据库**：不限制创建数据库的数量
3. **快速创建销毁**：用完即弃，不占用资源

具体实现是通过 MCP（Model Context Protocol），让 agent 可以创建和管理临时的 postgres 数据库。

#### 🔍 深度洞察

**记忆问题是 agent 从「玩具」变成「工具」的关键。** 如果 agent 只能处理单次对话，它的应用场景就非常有限。给它一个记忆层，它就能处理复杂的多步骤任务。

**临时数据库的设计很聪明。** 不需要一个持久的大数据库，而是每次任务创建临时数据库，用完就销毁。这样既解决了记忆问题，又不会积累垃圾数据。

**这个方向值得持续关注。** 记忆层正在成为 agent 的标配，未来会有更多类似的解决方案出现。

#### 🎯 行动建议

- 如果你做 agent 开发，认真研究这篇文章的方案
- 考虑给你的 agent 加一个记忆层，哪怕是最简单的版本
- 关注 MCP 协议的发展，这可能是 agent 记忆的标准方案

### 5. Speed vs Smarts：编程 agent 的取舍

**📌 原文信息**
- **标题**: [Speed vs smarts for coding agents?](https://dev.to/ben/speed-vs-smarts-for-coding-agents-3h)
- **来源**: Dev.to - Ben Halpern（DEV 创始人）
- **时间**: 2026-03-26
- **热度**: ❤️ 40 | 💬 21

![cover](https://media2.dev.to/dynamic/image/width=1000,height=420,fit=cover,gravity=auto,format=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F5v9k75ja7leded0xzzxu.png)
*文章封面*

#### 📖 背景介绍

DEV 创始人 Ben Halpern 发起讨论：编程 agent 应该追求快还是聪明？他好奇大家对这两种模式的偏好。

#### 💡 核心内容

两种模式：

1. **快速模式**：快速生成代码，但可能有问题
2. **聪明模式**：深思熟虑，生成的代码更可靠，但慢

评论区有人说：验证想法时用快速模式，正式开发时用聪明模式。

#### 🔍 深度洞察

**这个问题没有标准答案，取决于场景。** 如果你在快速原型阶段，快比完美重要。如果你在维护生产代码，稳定比快重要。

**最有价值的观点是：根据阶段选择。** 不要一味追求快，也不要一味追求完美。不同阶段有不同的最优解。

**agent 产品可能会走向「混合模式」。** 先快速生成，再深度优化。或者给用户选择权，让用户决定要快还是好。

#### 🎯 行动建议

- 用 AI 写代码时，先想清楚你的目标是什么
- 原型阶段可以多依赖 AI 的快速模式，但最终代码要人工审查
- 如果你开发 agent，考虑提供「快/好」的切换选项

### 6. 网站细节决定完成感

**📌 原文信息**
- **标题**: [The Small Details That Make a Website Feel Finished](https://dev.to/hadil/the-small-details-that-make-a-website-feel-finished-and-quietly-improve-accessibility-4jkp)
- **来源**: Dev.to
- **时间**: 2026-03-26
- **热度**: ❤️ 102 | 💬 15

![cover](https://media2.dev.to/dynamic/image/width=1000,height=420,fit=cover,gravity=auto,format=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fvq84fje9uqe06kmknaer.png)
*文章封面*

#### 📖 背景介绍

作者分享了一个观点：网站「能用」和「完成」之间，差的是细节。很多网站功能都有了，但总觉得差点意思，问题就出在细节上。

#### 💡 核心内容

文章列出了容易被忽略的细节：

1. **加载动画**：告诉用户「正在处理」，而不是卡住
2. **过渡效果**：页面切换的平滑过渡
3. **无障碍标签**：让屏幕阅读器能读懂
4. **焦点状态**：键盘导航时的视觉反馈
5. **错误提示**：友好的错误信息，而不是冷冰冰的错误代码

#### 🔍 深度洞察

**细节影响的是「信任感」。** 一个网站如果加载动画都没有，用户会觉得「这网站是不是崩了？」一个网站如果焦点状态都没有，用户会觉得「这网站是不是不专业？」

**无障碍不只是道德问题，是产品问题。** 很多开发者觉得无障碍是「加分项」，但对于有障碍的用户来说，这是「能不能用」的问题。

**细节是「专业感」的来源。** 你和一个资深前端的差距，往往不在于你会不会写代码，而在于你有没有关注这些细节。

#### 🎯 行动建议

- 做完网站后，对照这篇文章检查一遍
- 养成写无障碍标签的习惯，不是事后补救
- 用键盘导航一遍你的网站，看看哪里不顺畅

楼主还在用上古时代的 Intel iMac，感觉编译东西就卡爆了。问 M 芯片是不是真的很强，要不要等 M5。

#### 💡 核心内容

评论区一致回答：强，非常强。

1. **编译速度**：M 系列芯片的编译速度远超同价位的 Intel
2. **续航**：不插电也能高性能工作
3. **发热**：几乎不发热，风扇很少转
4. **噪音**：安静，几乎没有声音

#### 🔍 深度洞察

**从 Intel Mac 换到 M 系列，体验差距是「天翻地覆」级别的。** 不是快一点，是快很多。续航、发热、噪音全方位提升。

**要不要等下一代？看你的需求紧迫程度。** M3/M4 已经很强了，如果你现在的工作设备影响效率，就别等了。如果不急，可以等 M5，但永远有下一代。

**内存是短板。** M 系列芯片的内存不能升级，买的时候要考虑未来几年的需求。

#### 🎯 行动建议

- 如果你的 Intel Mac 已经影响工作效率，赶紧换，别等下一代
- 买的时候内存选大一点，16GB 是起步，做开发建议 24GB+
- 根据需求选型号：轻度办公 Air，重度开发 Pro
