1. V2EX 热议：你用 AI 赚到过钱吗？#

📌 原文信息

  • 标题: 你用 AI 赚到过钱吗 怎么赚的？
  • 来源: V2EX - 问与答板块
  • 时间: 2026-03-29
  • 热度: 47 条回复

V2EX 问与答板块图标

📖 背景介绍#

楼主在 V 站发帖，说现在刷站发现「所有东西都在跟 AI 沾边」，所以很想知道有没有人真的用 AI 赚到过钱，哪怕是一块钱。这个问题的背景是：AI 热潮持续了这么久，真正落地的变现案例到底有多少？

💡 核心内容#

评论区有几个真实的变现案例分享：

  1. AI 代写：有人接商业文案代写，用 AI 生成初稿再人工润色，效率翻三倍
  2. AI 绘图接单：有人做定制头像、logo 设计，用 Midjourney 或 Stable Diffusion 出图
  3. AI 数据分析：帮企业做数据分析报告，用 AI 处理数据生成洞见
  4. AI 工具开发：有人开发 AI 辅助工具卖订阅或服务

🔍 深度洞察#

赚钱的不是 AI 本身，而是用 AI 解决问题的能力。 会用 AI 生成文字不赚钱，但知道谁需要文字、需要什么样的文字，这个信息差才值钱。

AI 是工具，不是生意。 很多人把 AI 当成「赚钱机器」，但真正的赚钱逻辑是：发现需求 → 用 AI 高效解决 → 收费。AI 只是中间的工具环节。

门槛在降低，竞争在加剧。 当所有人都会用 AI 生成内容时，单纯的「AI 生成」就没有溢价了。差异化在于：懂行业、懂需求、能交付。

🎯 行动建议#

  • 如果你会编程，可以考虑做 AI 工具或插件，解决具体场景问题
  • 如果你不擅长技术，可以研究 AI 在自己行业的应用，比如 AI 辅助写作、AI 辅助设计
  • 关注「AI + 传统行业」的交叉点，那里机会更多

Google Sheets 数据可视化

Show HN 上有人做了个工具，把 Google Sheets 变成 CRUD 后端。写几行代码就能让你的「vibe coding」项目有数据库了。

对于副业开发者来说，这东西挺实用。有时候你想快速做个小产品，不想搭数据库、不想写 API，直接用 Google Sheets 就能搞定。

💡 核心内容#

核心功能：

  1. Google Sheets 作为数据库：直接用 Sheets 存数据，不用自己搭服务器
  2. CRUD API：提供创建、读取、更新、删除的 API
  3. 快速原型：几分钟就能给项目加上数据存储
  4. 成本低：Google Sheets 免费，不用付数据库费用

🔍 深度洞察#

这是「快速验证」的好工具。 做副业时，你不确定想法能不能赚钱，不想花时间搭基础设施。用这个工具，你可以快速上线验证，等验证成功了再迁移到正经数据库。

「vibe coding」的概念在流行。 很多人做项目不是为了赚钱，是为了好玩、为了验证想法。这类工具降低了做项目的门槛。

从 Sheets 迁移到正经数据库是个自然的成长路径。 很多成功的产品都是从简单方案开始的，然后根据需求逐步升级。

🎯 行动建议#

  • 等产品验证成功后，再考虑迁移到专业数据库

5. Agent 记忆问题：能思考但记不住#

📌 原文信息

  • 标题: your agent can think. it can’t remember.
  • 来源: Dev.to - ghost 团队
  • 时间: 2026-03-25 发布
  • 热度: ❤️ 147 | 💬 45 | 阅读时间 7 分钟

ghost 团队文章封面

📖 背景介绍#

ghost 团队写了一篇深度文章，讨论 AI agent 的核心瓶颈：它们能思考，但记不住。这个问题很现实：你让 agent 做一个多步骤任务，它做着做着就忘了前面做过什么。

💡 核心内容#

文章提出的解决方案：

  1. 临时 postgres 数据库：给 agent 一个即时、临时的数据库
  2. 无限数据库：不限制创建数据库的数量
  3. 快速创建销毁：用完即弃，不占用资源
  4. MCP 协议：通过 Model Context Protocol 实现记忆管理

🔍 深度洞察#

记忆问题是 agent 从「玩具」变成「工具」的关键。 如果 agent 只能处理单次对话，它的应用场景就非常有限。给它一个记忆层，它就能处理复杂的多步骤任务。

临时数据库的设计很聪明。 不需要一个持久的大数据库，而是每次任务创建临时数据库，用完就销毁。这样既解决了记忆问题，又不会积累垃圾数据。

MCP 正在成为标准协议。 Model Context Protocol 可能会成为 agent 记忆的标准方案，值得关注。

🎯 行动建议#

  • 如果你做 agent 开发，认真研究这篇文章的方案
  • 考虑给你的 agent 加一个记忆层，哪怕是最简单的版本
  • 关注 MCP 协议的发展，这可能是 agent 记忆的标准方案

6. Prompt Engineering 指南：如何提问改变一切#

📌 原文信息

  • 标题: Engenharia de Prompt: Por Que a Forma Como Você Pergunta Muda Tudo
  • 来源: Dev.to - He4rt Developers（葡萄牙语）
  • 时间: 2026-03-23 发布
  • 热度: ❤️ 128 | 💬 6 | 阅读时间 7 分钟

Prompt Engineering 指南封面

📖 背景介绍#

这是一篇葡萄牙语的 Prompt Engineering 指南，解释为什么提问的方式会改变 AI 的输出。虽然是葡萄牙语，但核心概念是通用的。

💡 核心内容#

关键要点：

  1. 提问方式决定输出质量：同一个问题，不同的提问方式会得到完全不同的答案
  2. 上下文很重要：给 AI 足够的上下文，它会理解你的意图
  3. 迭代优化：第一次提问不完美，可以追问和调整
  4. 结构化提问：用结构化的方式提问，输出也更结构化

🔍 深度洞察#

Prompt Engineering 是用 AI 的基础技能。 不是编程技能，是沟通技能。你能不能清晰表达需求，直接影响 AI 的输出质量。

很多人低估了提问的艺术。 随便扔一个问题给 AI，得到的答案也随便。花时间组织提问，得到的答案更有价值。

这个技能不限于 AI。 和人沟通也一样，能不能清晰表达需求，直接影响结果。Prompt Engineering 其实是在训练我们更好地表达。

🎯 行动建议#

  • 练习结构化提问：先描述背景，再描述需求，最后描述期望的格式
  • 第一次不满意时，追问和调整，而不是重新开始
  • 把好的提问方式记录下来，下次直接用

实验室手套可能是微塑料来源

📖 背景介绍#

密歇根大学研究发现，科学家用来研究的丁腈手套本身可能就是微塑料来源，导致研究数据被污染。

💡 核心内容#

关键发现：

  1. 手套本身就是微塑料：丁腈手套和乳胶手套都是塑料制品
  2. 污染研究数据：研究微塑料时，手套本身掉落的微塑料可能被误认为是样本
  3. 数据可能偏高：很多微塑料研究的数据可能因为手套污染而偏高
  4. 需要改进方法：科学家需要改进采样方法，避免手套污染

🔍 深度洞察#

这新闻有点讽刺。 研究微塑料的人，手套本身就是微塑料来源。科学实验里的「盲点」有时候就是这么意外。

科学研究的「系统性错误」很难发现。 很多人假设手套是干净的，但实际上手套本身就是塑料。这种假设很难被质疑，因为它是「常识」。

研究方法需要持续改进。 科学不是一次性的，而是持续改进的过程。发现手套污染后，需要改进采样方法。

🎯 行动建议#

  • 如果你做研究，注意潜在的系统性错误
  • 不要假设「标准工具」是完美的

  2. 无限循环：爬虫进去后会被困在无限链接循环中
